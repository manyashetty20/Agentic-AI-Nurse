Project Components Overview

  • Frontend (UI - Chat): HTML/CSS/JS (`index.html`). Provides the instant, hard-coded interview logic, Voice/Text input, and final report display.
  • Frontend (UI - Monitor): Web Framework: Streamlit (`app.py`). Used to build the entire interactive user interface, data display, and styling.
  • Frontend Tools: API Communication: Requests (Used in your `make_request` function to call the FastAPI backend). Data Manipulation: Pandas (Used to format data for display).
  • Backend (Main API): Web Framework: FastAPI (`main.py`). Used to build all API endpoints, including `/inventory/`, `/roster/generate`, and `/protocols/ask/`. ASGI Server: Uvicorn.
  • Backend Core Tools: Data Validation: Pydantic. Database: File-Based JSON (Used to manage `inventory.json`, `roster.json`, etc.). Scheduling Logic: Python `random` & `datetime` libraries.
  • API/Comm: API Communication: CORS (`Cross-Origin Resource Sharing`) middleware is crucial for frontend/backend communication. Environment Variables: python-dotenv.
  • AI Service (LLM): Model: Llama 3.3 70B (via Groq API). Tooling: Groq API (Used via the `groq` library).
  • AI Technique: RAG (Retrieval-Augmented Generation). Used to augment the AI's prompt with specialized protocol information extracted via **PyPDF2** from protocol files.
